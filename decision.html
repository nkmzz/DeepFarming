AI 필요성/사업 판단

이 프로젝트를 해야 하는지, AI가 진짜 필요한지, 1평 테스트와 대규모 목표를 어떻게 분리해서 봐야 하는지 정리한 판단 페이지.

Decision · Why This Project

이 일을 해야 하는지에 대한 고찰

핵심 질문은 “AI가 멋져 보이냐”가 아니라, 제어 대상이 예측 불가능한가, 데이터가 충분히 쌓이는가, 실제로 돈이나 시간을 아낄 수 있는가다.

현재 결론: 1평 테스트에서는 AI가 핵심이 아닐 수 있다. 하지만 대규모 스마트팜을 목표로 보면 AI는 장기적인 차별화 포인트가 된다.
구분판단이유
1평 테스트/MVPAI 필요성 낮음자동급수, 조명, 팬, 센서 기록은 룰 기반 제어로 충분
소형 파일럿낮음~중간사진 기록, 타임랩스, 잎면적 추출 정도는 OpenCV/약한 AI로 가능
수십~수백 평중간구역별 편차, 품질 균일도, 작업자 동선, 수확 예측이 중요해짐
수천 평 이상/공장형높음에너지비, 병해 조기감지, 수확량 예측, 레시피 최적화가 돈으로 연결
완전 자율 온실매우 높음사람 재배자의 경험 판단을 시스템화해야 함

정리하면, AI는 처음부터 펌프를 켜고 끄는 메인 제어기가 아니라, 대규모 운영에서 생육 상태를 읽고 예측하고 최적 조건을 추천하는 상위 판단 레이어로 들어가야 한다.

Scope Guardrail

이번 논의 반영 기준

가격, 하드웨어, 딥러닝, 수경 적합성은 아직 확정값이 아니라 판단 프레임으로 둔다.

가격/매출

현재는 실제 단가를 모르는 상태이므로, 1평 월매출은 확정 예측이 아니라 소형 가게 납품 가능성을 보는 러프 시나리오로만 둔다.

하드웨어

Raspberry Pi는 확정 결정이 아니다. Jetson, Mini PC, 스마트폰/USB 카메라와 함께 후보로 두고 비교한다.

AI/딥러닝

오늘 정리본이 기존 AI 심화 자료를 대체하지 않는다. 초기/중기/장기 적용 범위를 분리해 관리한다.

수경 적합성

마이크로그린은 저면관수 파일럿에는 좋지만, 양액 최적화 AI 기준 작물로는 상추와 분리해서 논의한다.

Differentiation Update

사업적·기술적 차별화 포인트 요약

무순/마이크로그린은 재배 자체가 쉬워 보일 수 있으므로, 차별화는 자동화 난제에 둔다.

기술 가설

무토양 매트 + 저면관수 + 트레이 무게 기반 수분 제어 + 급수/배수/환기 시퀀스가 핵심 후보. 상토는 기준군, 무토양 매트는 자동화 목표 후보로 둔다.

사업 가설

대량 도매가 아니라 소형 가게에 신선한 무순/마이크로그린을 반복 납품하는 운영 리듬을 검증한다. 4일 수확 가능성은 회전율 관점에서 중요한 검증 포인트다.

이 차별화가 실제로 남들도 어려워하는 지점인지는 아직 미검증이다. 이후 시장·현장·제품·특허 조사를 통해 검증한다. 상세 차별화 페이지 보기 →
Reality Check

지금 단계에서는 AI가 핵심이 아닐 가능성이 큼

대부분의 스마트팜 기본 동작은 센서 측정 → 기준값 비교 → 펌프/팬/조명/밸브 제어다. 이 영역은 AI보다 자동제어, IoT, PLC/MCU에 가깝다.

우선순위필요한 것AI 필요 여부
1온도/습도/수위/조도 측정불필요
2펌프/조명/팬 자동 제어불필요
3데이터 저장/그래프/알람불필요
4사진 기록/타임랩스불필요
5잎 면적/색상 자동 추출약한 AI 또는 OpenCV 가능
6병해/곰팡이 자동 감지AI 있으면 좋음
7수확일/수확량 예측데이터 쌓인 뒤 AI 가능
8광량·급수·양액 최적화 추천장기적으로 AI 가능
9완전 자율 재배고급 AI/제어 영역

룰 기반으로 충분한 예

습도 > 85% → 팬 ON
수위 낮음 → 펌프 OFF + 알람
조명 16시간 경과 → LED OFF
온도 > 28℃ → 환기팬 ON
pH/EC 범위 이탈 → 알람

자동차 제어와 비슷한 점

전압 UVLO/OVP, 온도 derating, PWM dimming 같은 명확한 조건은 AI보다 명시적 제어 로직이 더 안전하고 싸고 예측 가능하다.

초기 타이틀 조정

초기에는 “AI 스마트팜”보다 “데이터 기록형 자동 스마트팜” 또는 “소형 재배장 자동화 + 생육 데이터 수집 시스템”이 현실적이다.

Where AI Actually Matters

AI가 의미 있는 구간

AI는 직접 제어보다, 사람이 눈으로 판단하던 것을 대신하거나 많은 조건 중 최적점을 찾을 때 가치가 커진다.

영역AI 가치지금 필요성
잎 면적 측정매일 사진에서 생장 속도 자동 추출낮음~중간
색 변화 감지황화, 스트레스, 광 부족 추정중간
곰팡이/병반 감지사람이 놓치는 초기 이상 탐지중간
수확 시점 예측며칠 뒤 수확 가능한지 예측데이터 쌓인 뒤
수확량 예측사진+환경 → 중량 예측데이터 쌓인 뒤
조건 추천광량/급수/온도/양액 조합 추천장기 과제
자율 재배사람 설정 없이 전체 최적화고급 연구/상용 대형팜 영역

AI의 1차 가치는 “펌프 제어”가 아니라 작물 상태를 읽고, 구역별 편차를 찾고, 수확량·수확일·품질을 예측하는 것이다.

Scale Changes Everything

1평 테스트와 대규모 목표는 결론이 다름

1평은 실험실이고, 목표가 대규모 스마트팜이면 AI는 장기적으로 차별화 포인트가 맞다.

구분소형 테스트대규모 목표
센서몇 개 센서구역별 센서망
카메라1~2대랙/라인/이동형 카메라
제어Arduino/ESP32PLC/산업용 I/O/RS485/Ethernet
AI 연산없어도 됨Edge AI + 중앙 서버
데이터CSVDB + Batch ID + 이미지 저장소
제어 방식타이머/룰룰 + 예측 + 최적화
목표재배 성공균일 품질, 수확 예측, 비용 절감

구역별 편차 감지

A구역은 잘 자라는데 B구역 잎색이 연하거나, C구역 가장자리만 마르거나, D구역에 곰팡이 초기 징후가 있을 때 AI 비전이 가치가 생긴다.

수확량/수확일 예측

며칠 뒤 수확 가능한지, 몇 kg 나올지, 상품률이 얼마나 될지 예측하면 납품 일정·작업자 동선·출하 계획이 가능해진다.

에너지 최적화

LED, 냉난방, 제습, 공조 비용이 커지면 전기요금이 사업성을 좌우한다. 이때 예측/최적화 모델의 가치가 커진다.

재배 레시피 최적화

PPFD, DLI, 광질, 주야간 온도, VPD, CO₂, pH/EC, 급수, 재식밀도, 수확기준을 조건→품질→수익으로 연결해야 한다.

Architecture

스마트팜 AI는 위쪽 판단 레이어

AI가 아래쪽 제어를 대체하는 구조가 아니라, 자동제어 위에서 생육 판단·예측·최적화를 수행하는 구조가 맞다.

레이어역할기술AI 필요
L1. 안전/구동펌프, 팬, 밸브, LED, 히터 직접 제어MCU, PLC, 릴레이, MOSFET없음
L2. 환경 제어온도/습도/CO₂/광량/pH/EC 유지PID, 룰 기반, 시퀀스 제어거의 없음
L3. 모니터링사진, 센서, 수확량, 이상 상태 기록DB, 대시보드, 알람약간
L4. 생육 해석잎 면적, 색, 균일도, 병해, 수확시점 판단Vision AI, segmentation, detection필요
L5. 예측수확량, 수확일, 품질, 불량률 예측ML, 시계열 모델필요
L6. 최적화광량/양액/온습도/에너지비 최적화MPC, 강화학습, 디지털트윈고급 AI
L7. 자율 운영사람이 설정하지 않아도 재배 전략 조정AI + 제어 + 전문가 규칙매우 필요

한 줄 정리: AI는 스마트팜의 엔진이 아니라, 나중에 붙는 분석/판단 레이어다. 처음 엔진은 센서·제어·기록 시스템이다.

그래서 Jetson도 당장 필수는 아님

Jetson은 스마트팜 메인 제어기가 아니라 비전 AI 노드로 보는 게 맞다.

Final Judgement

최종 판단

소형 테스트와 대규모 목표를 분리하면 방향이 명확해진다.

틀린 방향

  • AI가 온도·습도·물·조명을 직접 제어한다
  • Jetson 하나 붙이면 AI 스마트팜이다
  • 처음부터 자율재배를 만든다
  • 데이터 없이 최적화부터 한다

맞는 방향

  • 기본 제어는 룰/PID/PLC가 한다
  • AI는 작물 상태를 읽고 예측한다
  • AI는 에너지·수확량·품질의 최적 조건을 추천한다
  • 대규모에서는 AI가 사람 재배자의 판단을 시스템화한다
소형 테스트에서는 AI가 필요 없다. 하지만 대규모 사업을 목표로 하면 AI는 필요하다. 단, AI는 하드웨어 제어기가 아니라 “생육 판단 + 예측 + 최적화 엔진”으로 들어가야 한다.

따라서 현재 프로젝트는 “AI 모듈 중심”이 아니라 “스마트팜 하드웨어 + 데이터/AI 확장 아키텍처”로 정리하는 것이 맞다.
